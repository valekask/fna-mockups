FNA-16209: Explore Widget — Layout Settings

UX Proposal: Separate "Layout Settings" panel (analogous to ILO "Analysis Settings")
Problem

The Explore Widget's existing Settings panel is dense with technical parameters. Users need a fast, intuitive way to configure Layout Algorithm and Priority Metrics without scrolling through advanced options.

Proposed Solution

Add a dedicated "Layout Settings" button in the widget toolbar that opens a separate side panel — exactly like ILO's "Analysis Settings" button. This keeps quick config separate from advanced settings, and provides an empty-state prompt when layout is not configured.

Existing Pattern — ILO "Analysis Settings"

Precedent in our codebase
This proposal reuses the exact same architecture already proven in ILO monitoring. The "Layout Settings" button mirrors the "Analysis" button pattern — same toolbar placement, same sidenav mechanism, same empty-state UX.
Aspect ILO "Analysis" (existing) Explore "Layout Settings" (proposed)
Toolbar button "Analysis" icon button "Layout Settings" icon button
Reload button "Reload analysis" (data-related) N/A — layout is a visual config, no reload needed
Side panel Opens via SidenavService as ILO_TEMPLATE_SETTINGS Opens via SidenavService as new type (e.g. LAYOUT_SETTINGS)
Empty state "The chart widget can't be created — Analysis configuration is not set up" "The Explore widget can't be created — Layout configuration is not set up"
Quick-open button Red "Analysis Settings" button in empty state Red "Layout Settings" button in empty state
Purpose Configure ILO template (data source & aggregation) Configure layout algorithm & priority metric
graph-header inputs showIloTemplateSettings, hasIloTemplateSettings Reuse same inputs OR add dedicated showLayoutSettings
Benefit: By reusing the existing graph-header + SidenavService + graph-empty-message infrastructure, implementation requires minimal new components — mostly wiring and the Layout Settings panel content itself.
